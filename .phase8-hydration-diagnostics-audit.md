## Blank Step Detection & Diagnostics

### Current Detection Points

**In _prepareContext (line 465-483):**
```javascript
const currentDescriptor = this.steps[this.currentStepIndex];  // Could be undefined!
const currentPlugin = currentDescriptor
  ? this.stepPlugins.get(currentDescriptor.stepId)
  : null;

const stepData = currentPlugin
  ? await currentPlugin.getStepData(context).catch(() => ({}))  // Catches errors!
  : {};
```

**Blank Step Conditions:**
1. `currentDescriptor` is undefined (no step at index)
2. `currentPlugin` is null (plugin not loaded for step ID)
3. `getStepData()` throws error (step data loading failed)
4. HTML template is null (work surface can't render)

---

### Diagnostic Rules

**Rule 8.1: Detect Missing Steps**
```javascript
if (!currentDescriptor) {
  diagnostics.push({
    level: 'error',
    code: 'MISSING_STEP_DESCRIPTOR',
    message: `No step descriptor at index ${this.currentStepIndex}`,
    recovery: 'Reset to first step',
  });
}
```

**Rule 8.2: Detect Missing Plugins**
```javascript
if (currentDescriptor && !currentPlugin) {
  diagnostics.push({
    level: 'error',
    code: 'MISSING_STEP_PLUGIN',
    message: `No plugin for step ${currentDescriptor.stepId}`,
    recovery: 'Reload step plugins',
  });
}
```

**Rule 8.3: Detect Blank HTML Templates**
```javascript
if (!workSurfaceHtml && currentPlugin) {
  diagnostics.push({
    level: 'warning',
    code: 'BLANK_TEMPLATE',
    message: `No template rendered for ${currentDescriptor.stepId}`,
    recovery: 'Show placeholder UI',
  });
}
```

**Rule 8.4: Detect Invalid Step Data**
```javascript
if (!stepData || typeof stepData !== 'object') {
  diagnostics.push({
    level: 'warning',
    code: 'INVALID_STEP_DATA',
    message: `Step data not an object: ${typeof stepData}`,
    recovery: 'Use empty object {}',
  });
}
```

## Hydration Authority Standards

### Initialization Invariants

**Invariant 1: Always Have a Current Step**
```
Rule: this.currentStepIndex must point to valid step index
Guarantee: After _initializeFirstStep(), currentStepIndex is valid
Enforcement: Check _initializeFirstStep() doesn't throw unhandled
```

**Invariant 2: Steps Array Never Empty**
```
Rule: this.steps.length >= 1 after _initializeSteps()
Guarantee: At least the intro step or first chargen step exists
Enforcement: Log count, verify > 0
```

**Invariant 3: Current Plugin Matches Current Step**
```
Rule: If steps[currentStepIndex] exists, stepPlugins has entry for its ID
Guarantee: currentPlugin = stepPlugins.get(steps[currentStepIndex].stepId) always valid
Enforcement: Check before calling currentPlugin methods
```

**Invariant 4: Mentor State Always Defined**
```
Rule: this.mentor is never undefined
Guarantee: _initializeMentorState() always completes
Enforcement: Fallback to default mentor if load fails
```

## Blank Step Recovery Strategies

### Strategy 1: Fallback to First Valid Step
```javascript
if (!currentDescriptor || currentStepIndex >= steps.length) {
  console.warn('Current step invalid, resetting to 0');
  this.currentStepIndex = 0;
  currentDescriptor = this.steps[0];
}
```

### Strategy 2: Show Placeholder UI
```javascript
if (!workSurfaceHtml) {
  workSurfaceHtml = `
    <div class="prog-work-placeholder">
      <p>Step content loading...</p>
      <button onclick="location.reload()">Retry</button>
    </div>
  `;
}
```

### Strategy 3: Skip Unloadable Plugins
```javascript
if (!currentPlugin) {
  console.error(`Step plugin not found: ${stepId}`);
  // Continue with null plugin
  // Show blank details panel
  detailsPanelHtml = null;
}
```

### Strategy 4: Reload Step Plugins
```javascript
if (missingPlugins.length > 0) {
  console.log('Reloading step plugins...');
  await this._initializeSteps();  // Re-discover plugins
}
```

## Expected Hydration Outcome

### Successful Hydration

```
✅ Constructor completes
✅ _initializeSteps() discovers all steps
✅ _initializeFirstStep() loads step 0
✅ _prepareContext() hydrates with valid state
✅ No diagnostics OR only info-level diagnostics
✅ First render displays work surface
```

### Partial Failure Recovery

```
⚠️ Plugin missing for step "skills"
  → Skip that step's details rendering
  → Continue with rest of progression
  → Show warning in console

⚠️ Template fails to render for "class"
  → Show placeholder UI
  → Allow user to continue (maybe data loads on next render)
  → Log warning diagnostic
```

### Total Failure Handling

```
❌ No steps discovered
  → Show error to user
  → Offer reload button
  → Log critical diagnostic

❌ First step fails to initialize
  → Show error notification
  → Prevent progression
  → Log error diagnostic with stack trace
```
