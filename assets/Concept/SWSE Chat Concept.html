SWSE — Enhanced Chat Messages · Concept Pass

A single unified card shell that adapts by category — built on the existing --swse-* theme tokens, holo-roll.hbs, and the SWSEChat.postRoll() pipeline. Every card flows from the data already produced by the roll engines — actor, action, formula, total, DC, target, success — without introducing new roll math.

System @ a glance

  • One shell: .swse-chat-card + a category modifier (--skill, --attack, --damage, --save, --force, --dark, --initiative, --ability, --dialogue).
  • Category sets a single --rail token; everything else (chip, total color, button glow) derives from it via color-mix().
  • Outer chrome and palette pull straight from tokens.css + chat-surfaces.css — no hardcoded hex anywhere the user can change.
  • Slot map for holo-roll.hbs: title→action, category→rail, context.dc/passed→outcome, context.rerollOptions→actions, weaponVisual→.swse-roll-weapon-visual.
  • Responsive breakpoint at ~360 px collapses the total / breakdown grid to a single row.

Weapon color

  • Attack & damage cards adopt the player's chosen weapon color end-to-end — saber crystal, blaster bolt, vibroblade plasma, etc. Set inline: style="--rail: {{weaponVisual.colorHex}}" on the card root. WeaponVisualProfileResolver already produces this value; the existing holo-roll.hbs just needs to lift it from the beam pill to the card root.
  • Rail · ribbon · total · button glow · weapon beam all follow one variable. No second resolver needed.

Runtime notes

  • No new roll pipelines — every card consumes existing buildHoloRollData() output.
  • Inline action buttons reuse the existing swse-skill-reroll-btn / swse-attack-reroll-btn / swse-roll-damage handlers; only their chrome changes.
  • Foundry v13: rendered with foundry.applications.handlebars.renderTemplate; card root keeps data-swse-chat-surface="roll" so the chat-surfaces.css bridge still selects it.
