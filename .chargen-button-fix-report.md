# Chargen Button Mismatch — Investigation & Fix Report

**Date:** March 23, 2026
**Status:** ✅ FIXED
**Issue:** Chargen button unresponsive on NPC and Droid sheets

---

## Problem Statement

Chargen button was not working when clicked on NPC and Droid character sheets. The button appeared in the UI but clicking produced no action.

---

## Root Cause Analysis

### The Structural Mismatch

Two button wiring systems were in conflict:

#### System 1: V2 Character Sheet (Active)
**File:** `templates/actors/character/v2/character-sheet.hbs`

```handlebars
{{#if isLevel0}}
  <button type="button" data-action="cmd-chargen" class="header-btn">Chargen</button>
{{else}}
  <button type="button" data-action="cmd-levelup" class="header-btn">Level Up</button>
{{/if}}
```

**Wiring:** `scripts/sheets/v2/character-sheet.js`
```javascript
const chargenButtons = html.querySelectorAll('[data-action="cmd-chargen"]');
chargenButtons.forEach(button => {
  button.addEventListener("click", async ev => {
    await launchProgression(this.actor);
  }, { signal });
});
```

**Status:** ✅ Works correctly
- Uses `data-action="cmd-chargen"` attribute
- Listener properly wired in character-sheet.js
- Only shows for level 0 actors (behind `isLevel0` check)

#### System 2: Persistent-Header Partial (Legacy)
**File:** `templates/partials/actor/persistent-header.hbs`

```handlebars
{{#unless chargenComplete}}
  <button class="character-generator">Chargen</button>
{{/unless}}
<button class="open-store">Shop</button>
<button class="talk-to-mentor">Mentor</button>
```

**Problem:** ❌ NOT WIRED
- Uses class names (`.character-generator`, `.open-store`, `.talk-to-mentor`)
- NO matching listeners in droid-sheet.js or npc-sheet.js
- Buttons appear but do nothing when clicked

### Sheet Affiliation

| Sheet | Template Used | Button Contract | Listener Status |
|-------|---------------|-----------------|-----------------|
| Character Sheet | V2 (character-sheet.hbs) | data-action | ✅ Wired |
| Droid Sheet | persistent-header.hbs | .class only | ❌ NOT wired |
| NPC Sheet | persistent-header.hbs | .class only | ❌ NOT wired |
| Vehicle Sheet | N/A | N/A | N/A |

## Solution Implemented

### Fix 1: Update persistent-header.hbs Button Markup

**Changed from:**
```handlebars
<button class="character-generator">Chargen</button>
<button class="open-store">Shop</button>
<button class="talk-to-mentor">Mentor</button>
```

**Changed to:**
```handlebars
<button type="button" data-action="cmd-chargen">Chargen</button>
<button type="button" data-action="cmd-store">Shop</button>
<button type="button" data-action="open-mentor">Mentor</button>
```

**Impact:** All three sheets (Character, Droid, NPC) now use consistent `data-action` attributes

---

### Fix 2: Add Listeners to droid-sheet.js

**File:** `scripts/sheets/v2/droid-sheet.js`
**Location:** _onRender method, after initiative controls (line ~288)

```javascript
/* ---- PROGRESSION FRAMEWORK BUTTONS (Chargen/Store/Mentor) ---- */

root.querySelector('[data-action="cmd-chargen"]')?.addEventListener("click", async (ev) => {
  ev.preventDefault();
  const { launchProgression } = await import("/systems/foundryvtt-swse/scripts/apps/progression-framework/shell/chargen-shell.js");
  await launchProgression(this.actor);
}, { signal });

root.querySelector('[data-action="cmd-store"]')?.addEventListener("click", async (ev) => {
  ev.preventDefault();
  const { launchProgression } = await import("/systems/foundryvtt-swse/scripts/apps/progression-framework/shell/chargen-shell.js");
  await launchProgression(this.actor, 'store');
}, { signal });

root.querySelector('[data-action="open-mentor"]')?.addEventListener("click", async (ev) => {
  ev.preventDefault();
  ui.notifications.info("Mentor interactions coming soon!");
}, { signal });
```

**File:** `scripts/sheets/v2/npc-sheet.js`

Same listener code as droid-sheet.js (3 buttons: cmd-chargen, cmd-store, open-mentor)

## Technical Details

### Import Path

The listeners use dynamic import to avoid circular dependencies:
```javascript
const { launchProgression } = await import(
  "/systems/foundryvtt-swse/scripts/apps/progression-framework/shell/chargen-shell.js"
);
```

This matches the pattern used in character-sheet.js and avoids coupling issues.

### Abort Controller Integration

Both listeners use the existing `{ signal }` from the AbortController:
```javascript
this._renderAbort = new AbortController();
const { signal } = this._renderAbort;
// ... later ...
button.addEventListener("click", handler, { signal });
```

This ensures old listeners are cleaned up when the sheet re-renders, preventing duplicate handlers.

### Store Action Parameter

The store button passes a second parameter:
```javascript
await launchProgression(this.actor, 'store');
```

This is consistent with the progression framework's ability to start at specific modes.

## Design Principles Enforced

✅ **Unified Button Contract:** All sheets now use `data-action` attributes for these high-level actions

✅ **Consistent Listener Pattern:** Droid and NPC sheets follow the same listener pattern as character-sheet.js

✅ **AbortController Integration:** Prevents duplicate event handlers across re-renders

✅ **Dynamic Imports:** Avoids circular dependencies in the module system

✅ **Graceful Fallbacks:** Mentor button on droid/NPC shows notification rather than crashing

## Related Code Paths

### Chargen Launch Path
```
droid/npc button click
  → [data-action="cmd-chargen"] listener
    → await launchProgression(this.actor)
      → ChargenShell.open(actor)
        → ProgressionShell.open(actor, 'chargen')
          → Full progression UI render
```

### Store Launch Path
```
store button click
  → [data-action="cmd-store"] listener
    → await launchProgression(this.actor, 'store')
      → ChargenShell.open(actor, 'store')
        → ProgressionShell.open(actor, 'chargen', { currentStep: 'store' })
```
