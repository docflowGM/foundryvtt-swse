## Implementation Details

### Rule 8.1: Missing Step Descriptor
**Detection Point:** `_prepareContext()` line 470
**Condition:** `!currentDescriptor` when accessing `this.steps[this.currentStepIndex]`
**Recovery:** `fallbackToFirstStep()` — reset to index 0
**UI Message:** "No step descriptor at index {idx} (total steps: {total})"
**Console:** Error level with red color

---

### Rule 8.2: Missing Step Plugin
**Detection Point:** `_prepareContext()` line 481
**Condition:** Step exists but `!this.stepPlugins.has(stepId)`
**Recovery:** `skipMissingPlugin()` — show null for details panel
**UI Message:** "No plugin for step \"{stepId}\""
**Console:** Error level, includes registered plugin list

---

### Rule 8.3: Blank HTML Template
**Detection Point:** `_prepareContext()` line 519
**Condition:** `!workSurfaceHtml` after render attempt
**Recovery:** Show placeholder HTML with "Loading..." and retry button
**UI Message:** "Work surface template is blank or null for step \"{stepId}\""
**Console:** Warning level

---

### Rule 8.4: Invalid Step Data
**Detection Point:** `_prepareContext()` line 498
**Condition:** `!stepData || typeof stepData !== 'object'`
**Recovery:** Use empty object `{}` as fallback (already done in code)
**UI Message:** "Step data is not a valid object: {type}"
**Console:** Warning level

## Invariant Validation

### Invariant 1: Always Have a Current Step
**Check:** `currentStepIndex` is valid integer within bounds
**Violation Message:** "Current step index is invalid or negative"
**Enforcement:** Checked in `HydrationValidator.validateInitialization()`

### Invariant 2: Steps Array Never Empty
**Check:** `this.steps.length > 0`
**Violation Message:** "Steps array is empty after initialization"
**Enforcement:** In `_initializeSteps()` line 355-362

### Invariant 3: Current Plugin Matches Current Step
**Check:** If step exists, plugin must exist for its ID
**Violation Message:** "No plugin for step \"{stepId}\""
**Enforcement:** Rule 8.2 detection in `_prepareContext()`

### Invariant 4: Mentor State Always Defined
**Check:** `this.mentor` is object type
**Violation Message:** "Mentor state is undefined or invalid"
**Enforcement:** In `HydrationValidator.detectMissingMentor()`

## Recovery Strategies

### Strategy 1: Fallback to First Step
```javascript
// Called when currentDescriptor is missing
this.currentStepIndex = 0;
currentDescriptor = this.steps[0];
// Continue rendering with first valid step
```

### Strategy 2: Show Placeholder UI
```html
<div class="prog-work-placeholder">
  <p>Step content is loading...</p>
  <button onclick="window.location.reload()">Retry</button>
</div>
```

### Strategy 3: Skip Unloadable Plugins
```javascript
// If plugin missing:
const plugin = null; // Don't crash, just skip details
detailsPanelHtml = null; // Show empty details panel
```

### Strategy 4: Reload Step Plugins
```javascript
// Async re-discovery of plugins
await app._initializeSteps();
// Plugins re-registered and available
```

## Known Limitations

1. **Diagnostic Persistence:** Diagnostics are logged to console but not persisted to file/database. For production debugging, consider adding diagnostic export.

2. **User Notifications:** Error notifications shown via `ui.notifications.error()`. On some setups this may not be visible if notifications are suppressed.

3. **Plugin Reload:** Strategy 4 (reload plugins) assumes plugins can be rediscovered. If plugin files are missing, reload will fail silently.

4. **Recovery Scope:** Fallback to first step resets currentStepIndex. User loses position but can manually navigate back via step chips.
