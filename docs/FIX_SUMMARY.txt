                     RENDER QUEUE FIX — SUMMARY REPORT
═══════════════════════════════════════════════════════════════════════════════

ROOT CAUSE
──────────
The progression shell's render guard discarded nested rerender requests during
active renders. When Species step clicked an item:
  1. Set shell.focusedItem = entry
  2. Await shell.mentorRail.speak() (async animation)
  3. Call shell.render() to hydrate the detail panel

If the shell was still considered "rendering", the critical detail hydration
rerender was BLOCKED and silently discarded, leaving the right rail empty.

Console evidence:
  "[ProgressionShell] Render called while already rendering — BLOCKED"

    if (this._isRendering) {
      console.warn("[ProgressionShell] ⚠️ Render called while already rendering — BLOCKED (loop prevention)");
      return this;
    }
    this._isRendering = true;
    this._renderCount++;
    console.log(`[ProgressionShell] RENDER START (#${this._renderCount}) position:`, this.position);
    const result = await super.render(...args);

  }

NEW:
  async render(...args) {
    // Render queue: if render is in progress, queue one pending rerender instead of blocking
    if (this._isRendering) {
      console.warn("[ProgressionShell] ⚠️ Render called while already rendering — QUEUEING (will retry after)");
      this._pendingRender = true;
      return this;
    }
    this._isRendering = true;
    this._renderCount++;
    console.log(`[ProgressionShell] RENDER START (#${this._renderCount}) position:`, this.position);
    try {
      const result = await super.render(...args);
      console.log(`[ProgressionShell] RENDER COMPLETE (#${this._renderCount}) position:`, this.position);
      return result;
    } finally {
      this._isRendering = false;
      if (this._pendingRender) {
        this._pendingRender = false;
        console.log(`[ProgressionShell] EXECUTE QUEUED RERENDER (#${this._renderCount + 1})`);
        queueMicrotask(() => this.render());
      }
    }
  }

CHANGES:
  • Line 425–428: Queue the rerender instead of blocking (set _pendingRender)
  • Line 436–449: Wrap in try/finally to ensure lock is always released
  • Line 444–448: After lock is released, execute queued rerender if flagged
  • Uses queueMicrotask() for safe async scheduling (standard browser API)
