## Standardized Rules

### 1. Scrollbar Styling

**Applied to:**
- All scrollable regions: mentor-rail, progress-rail, utility-bar, work-surface, details-panel, summary-panel

**Implementation:**
```css
/* Firefox */
scrollbar-width: thin;
scrollbar-color: rgba(80, 220, 255, 0.5) rgba(80, 220, 255, 0.1);

/* Webkit */
::-webkit-scrollbar { width: 8px; height: 8px; }
::-webkit-scrollbar-track { background: rgba(80, 220, 255, 0.05); border-radius: 4px; }
::-webkit-scrollbar-thumb {
  background: rgba(80, 220, 255, 0.5);
  border-radius: 4px;
  border: 2px solid transparent;
  background-clip: content-box;
}
::-webkit-scrollbar-thumb:hover { background: rgba(80, 220, 255, 0.7); }
```

**Result:**
- Consistent cyan-themed scrollbars across all platforms
- Thin 8px width doesn't overwhelm interface
- Hover effect provides visual feedback
- Matches holographic aesthetic established in Phase 3-4

### 2. Text Wrapping & Overflow

**Applied to:**
- `.prog-mentor__text` (mentor dialogue)
- `.prog-step__name` (progress step labels)
- `.utility-item` (utility bar content)
- `.work-content-text` (work surface text)
- `.details-text` (details panel text)

**Implementation:**
```css
word-wrap: break-word;
overflow-wrap: break-word;
hyphens: auto;
max-width: 100%;
min-width: 0;  /* Allow flex shrink */

/* For code elements */
word-break: break-all;
font-family: "Courier New", monospace;
```

**Result:**
- Consistent text handling across all rail regions
- Long words hyphenate intelligently
- Unbreakable words (URLs, code) break at container edge
- Text never creates unexpected layout expansion

**Result:**

- Prevents unexpected layout jumps

- Allows responsive adaptation on small screens

### 6. Accessibility Compliance

**Motion Preferences:**
```css
@media (prefers-reduced-motion: reduce) {
  ::-webkit-scrollbar-thumb {
    transition: none;
  }
}
```

**High Contrast Mode:**
```css
@media (prefers-contrast: more) {
  /* Increase border and scrollbar visibility */
  border-bottom-color: rgba(80, 220, 255, 0.5);  /* from 0.25 */
  scrollbar-thumb: rgba(80, 220, 255, 0.7);       /* darker */
}
```

**Keyboard Navigation:**
```css
:focus-visible {
  outline: 2px solid rgba(100, 220, 255, 0.8);
  outline-offset: -2px;
}
```

## Design Decisions

### Why Scrollbar-Gutter: Stable?
Prevents layout shift when scrollbar appears/disappears. Allocates space for scrollbar even when not visible, maintaining consistent text positioning.

### Why Min-Height: 300px for Content Columns?
Ensures readable minimum of ~15-20 lines of text at standard font sizes. Prevents accidental content truncation on reasonable window sizes.

### Why Separate Border Brightness per Rail?
Mentor rail (primary): Brighter glow (0.08 opacity)
Progress rail (secondary): Dimmer glow (0.05 opacity)
Utility bar (tertiary): Dimmest glow (0.03 opacity)
Visual hierarchy guides user attention through interface.

### Why Responsive Height Breakpoints?
- <800px: Single 140px mentor rail prevents squishing
- <600px: 120px mentor rail + smaller progress rail maintain usability
- Tested breakpoints based on common device heights
