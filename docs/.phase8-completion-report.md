## Code Architecture

```
progression-shell (main entry point)
├── _initializeSteps()
│   ├── Validates steps array non-empty (Invariant 2)
│   ├── Instantiates plugins with error handling
│   └── Reports errors to user
├── _initializeFirstStep()
│   ├── Checks descriptor and plugin exist
│   ├── Calls plugin.onStepEnter() with error handling
│   └── Shows error notification on failure
└── _prepareContext()
    ├── Creates HydrationDiagnosticsCollector
    ├── Rule 8.1: Detect missing descriptor + fallback
    ├── Rule 8.2: Detect missing plugin + skip
    ├── Rule 8.3: Detect blank template + placeholder
    ├── Rule 8.4: Detect invalid step data + empty object
    ├── Logs diagnostics to console
    └── Returns context with diagnostics for UI
        ├── diagnostics.blocking[] → Error banners
        ├── diagnostics.warning[] → Warning banners
        └── diagnosticsFull → JSON for debugging
```

## Error Message Audit

### Critical (Blocking) Messages
1. **"No progression steps available..."**
   - Shown when: Empty steps array
   - Severity: 🔴 Critical
   - User Action: Reload application

2. **"No step descriptor at index..."**
   - Shown when: Current step invalid after fallback fails
   - Severity: 🔴 Critical
   - User Action: Reload application

### Error Messages (High Priority)
3. **"No plugin registered for step..."**
   - Shown when: Plugin not found for current step
   - Severity: 🔴 Error
   - User Action: Check console, contact support

4. **"Failed to initialize step..."**
   - Shown when: Plugin.onStepEnter() throws
   - Severity: 🔴 Error
   - User Action: Reload application

### Warning Messages (Low Priority)
5. **"Work surface template is blank..."**
   - Shown when: Template doesn't render
   - Severity: 🟡 Warning
   - User Action: Click Retry button

6. **"Step data is not a valid object..."**
   - Shown when: getStepData() returns non-object
   - Severity: 🟡 Warning
   - User Action: Continue, details may be incomplete

## Known Issues & Workarounds

### Issue 1: Diagnostic Banners Don't Appear
**Cause:** partial include syntax not recognized by template engine
**Workaround:** Use handlebars include syntax: `{{#include "apps/progression-framework/diagnostic-banner"}}`
**Status:** May need adjustment based on Foundry version

### Issue 2: Error Notifications May Not Show
**Cause:** Notifications disabled in user's settings or browser
**Workaround:** Always check browser console for detailed messages
**Status:** Documented in console output

### Issue 3: Plugin Reload Strategy May Fail
**Cause:** Plugin files deleted or missing from system
**Workaround:** Manual reload of entire application required
**Status:** Falls back to showing error message
