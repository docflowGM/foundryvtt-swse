## Text Wrapping Implementation

### CSS Properties Applied

**Location:** `/styles/progression-framework/shell/rails.css` (`.prog-mentor__text`)

```css
/* Block-level text rendering */
display: block;
width: 100%;

/* Word breaking rules (cascading priorities) */
word-wrap: break-word;              /* Legacy support */
overflow-wrap: break-word;          /* Modern standard */
word-break: break-word;             /* Fallback */
hyphens: auto;                      /* Smart hyphenation */

/* Widow/orphan prevention */
text-align: justify;                /* Distribute words across line width */
min-height: 1.55em;                 /* At least one line of content */
max-width: 100%;
min-width: 0;                       /* Allow flex shrink */
```

### Behavior by Content Type

| Content Type | Behavior | Example |
|------------|-----------|---------|
| **Normal text** | Words wrap at space; long words hyphenate | "The mentor system has been fully integrated..." |
| **Long words** (>40 chars) | Hyphenate or break at syllables | "internationalization" → "international-ization" |
| **URLs/Email** | Force break if necessary | "https://example.com/very/long/path" |
| **Code/Technical** | Monospace, break-all mode | "`function.methodName()`" |
| **Punctuation** | Normal wrapping rules apply | "Do this: prepare, analyze, implement." |
| **Single word** | Wraps if exceeds container | Extremely long words break after ~728px |

### Hyphenation Rules
- **Enabled:** `hyphens: auto` allows language-appropriate hyphenation
- **Fallback:** If hyphenation unavailable, `word-break: break-word` forces break
- **Minimum:** Most browsers require ≥5 characters before hyphenating

## Overflow & Scrolling

### Dialogue Box Constraints

**Location:** `/styles/progression-framework/shell/rails.css` (`.prog-mentor__dialogue`)

```css
height: calc(100% - 12px);          /* Fits within mentor rail (160px) */
overflow-y: auto;                    /* Scroll when content exceeds height */
overflow-x: hidden;                  /* Never scroll horizontally */
scrollbar-gutter: stable;            /* Prevent layout shift when scrollbar appears */
```

**Scroll Behavior:**
- Content > 120px height: vertical scrollbar appears
- Scrollbar styled to match cyan holographic theme
- Scroll position preserved on partial rerender
- No horizontal scrollbar (content constrained to width)

### Scrollbar Styling

**Firefox:**
```css
scrollbar-width: thin;
scrollbar-color: rgba(80, 220, 255, 0.5) transparent;
```

**Webkit (Chrome, Safari, Edge):**
```css
::-webkit-scrollbar { width: 8px; }
::-webkit-scrollbar-track { background: rgba(..., 0.05); }
::-webkit-scrollbar-thumb {
  background: rgba(80, 220, 255, 0.5);
  border-radius: 4px;
  border: 2px solid transparent;
  background-clip: content-box;
}
::-webkit-scrollbar-thumb:hover {
  background: rgba(80, 220, 255, 0.7);
}
```

**Visual Result:**
- Thin, elegant scrollbar that doesn't overwhelm dialogue
- Cyan color matches holographic theme
- Hover effect indicates interactivity
- Smooth 2px padding creates visual separation

## Edge Case Handling

### 1. Very Short Content
**Test Case:** Single word, single line
```
Input:  "Wait."
Output: Single line, no scroll
```
**Handling:**
- `min-height: 1.55em` ensures consistent vertical spacing
- Idle state uses same rules, maintains alignment

### 2. Very Long Content
**Test Case:** Multi-paragraph dialogue
```
Input:  [500+ characters]
Output: Multiple lines with scroll indicator
```
**Handling:**
- Scrollbar appears automatically when height > 120px
- Text remains fully readable due to line-height: 1.55
- Scroll position maintained across updates

### 3. Long Unbreakable Words
**Test Case:** URLs, technical terms, made-up words
```
Input:  "Visit https://www.example.com/very/long/documentation/path"
Output: Word breaks at container edge, wrapped to next line
```
**Handling:**
```css
overflow-wrap: break-word;   /* Primary method */
word-break: break-word;      /* Fallback */
```
For `<code>` elements: `word-break: break-all` forces character-level break.

### 4. Special Characters & Unicode
**Test Case:** Emoji, diacritics, symbols, RTL text
```
Input:  "Well done! 🎉 You've completed the task. ✓"
Output: Renders correctly with proper line breaks
```
**Handling:**
- `overflow-wrap: break-word` respects Unicode boundaries
- Hyphens won't be inserted in emoji or symbol sequences
- RTL text flows naturally (browser handles bidi)

### 5. Idle State Transition
**Test Case:** Text changes from "Awaiting decision..." to multi-line response
```
Input:  Idle → Mentor.currentDialogue update
Output: Animation triggers, content expands, scroll updates
```
**Handling:**
- Both idle and active states use same text constraints
- Reveal animation applies only to non-idle content
- Scroll height recalculated automatically on content change
- CSS Grid/Flex parent properly contains expanded content

### 6. Whitespace Handling
**Test Case:** Multiple spaces, tabs, line breaks in content
```
Input:  "Text    with    extra    spaces"
Output: Extra spaces preserved, text wraps normally
```
**Handling:**
- `white-space: normal` (default) collapses consecutive spaces
- Line breaks (`\n`) rendered as spaces (Handlebars template level)
- Tab characters (`\t`) rendered as single space

## Design Decisions

### Why `display: block` instead of `inline-block`?
- Block allows proper width constraint to 100% of container
- Block respects overflow-wrap and word-break rules better
- Block allows min-width: 0 for proper flex child sizing

### Why `text-align: justify` for short content?
- Improves visual distribution across container width
- Prevents awkward ragged-right edges
- Only affects multi-word lines (single-word lines unaffected)

### Why `scrollbar-gutter: stable`?
- Prevents layout shift when scrollbar appears (improving UX)
- Scrollbar width reserved even when not visible
- Better alignment of text across static/scrollable states

### Why custom webkit scrollbar styling?
- Default scrollbar is thick and visually noisy
- Custom styling matches holographic cyan theme
- Hover effect provides interactive feedback
