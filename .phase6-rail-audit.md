## Identified Inconsistencies

### 1. Scrollbar Styling

**Current State:**
- Mentor dialogue: Custom styled cyan scrollbar (Firefox + Webkit)
- Progress rail: Default browser scrollbar
- Other regions: Default/unspecified

**Issue:** Visual inconsistency; doesn't communicate unified holographic interface

**Standardization Goal:** Extend styled cyan scrollbar to all scrollable regions

---

### 2. Text Wrapping & Overflow

**Current State:**
- Mentor dialogue: Phase 5 hardening applied (word-break, overflow-wrap, hyphens)
- Progress rail: Single-line items (white-space: nowrap)
- Work surface: Varies by step (card grids, forms, etc.)
- Details panel: Unknown

**Issue:** No unified text overflow/wrapping strategy across rails

**Standardization Goal:** Apply consistent text constraints and wrapping rules

---

### 3. Content Padding

**Current State:**
- Mentor rail: 8px 20px
- Progress rail: 8px 12px
- Utility bar: Unknown
- Content row columns: Unknown

**Issue:** Inconsistent spacing creates visual fragmentation

**Standardization Goal:** Define unified padding scheme (e.g., `var(--prog-space-md)`)

---

### 4. Border & Glow Styling

**Current State:**
- Shell frame: Cyan border + box-shadow glow
- Mentor rail: SVG frame + border-bottom + box-shadow
- Progress rail: Border-bottom + box-shadow
- Content row: Unknown
- Action footer: SVG frame (chip backgrounds)

**Issue:** Glow intensity and border styles vary; not unified

**Standardization Goal:** Define consistent border/glow rules for all rail divisions

---

### 5. Scroll Indicators

**Current State:**
- Mentor dialogue: Styled cyan scrollbar (clear indicator)
- Progress rail: Default scrollbar (minimal visibility)
- Other regions: Unknown or default

**Issue:** Users may not recognize when content is scrollable (Progress rail)

**Standardization Goal:** Add visual scroll indicators or styled scrollbars to all regions

---

### 6. Content Constraints

**Current State:**
- Mentor dialogue: Width ~728px, height ~90px (SVG frame)
- Work surface: Flex-grow (varies by step content)
- Details panel: Fixed or flex (unverified)
- Progress rail: Horizontal scroll (no constraint)

**Issue:** No unified max-width or height constraints for readable content

**Standardization Goal:** Define max-width for readable line length, min-height for visual balance
