# Cursor Authority Implementation — Cross-Surface Enforcement

**Date:** March 23, 2026
**Status:** ✅ COMPLETE
**Scope:** Extended SWSE cursor pack from app UI to main Foundry scene canvas/board

---

## Problem Statement

The SWSE custom cursor pack (futuristic SVG cursors) was working correctly on:
- ✅ Application windows (character sheet, inventory, etc.)
- ✅ Sidebar elements
- ✅ Progression framework shells
- ✅ Dialog boxes

But was **NOT** appearing on:
- ❌ Main scene canvas/board
- ❌ Placeables (tokens, walls, templates)
- ❌ Canvas interaction areas

**Root Cause:** CSS cursor rules were scoped only to `.app`, `.application`, `.progression-shell`, etc. The main board/canvas element (`#board`) was never targeted, so it defaulted to OS cursor.

---

## Solution Architecture

Two-layer cursor enforcement:

### Layer 1: CSS Rules (swse-cursors.css)
**Purpose:** Declarative cursor assignment for static UI elements
**Coverage:** App windows, buttons, text inputs, and now board/canvas

**New CSS Selectors Added:**
```css
body.game #board                    /* Board container */
body.game #board canvas             /* Canvas element */
body.game .canvas                   /* Canvas wrapper */
body.game #board .canvas-stage      /* Pixi stage */
body.game .placeable                /* Token/wall/etc objects */
body.game .placeable:hover          /* Interactive placeable states */
body.game [draggable="true"]         /* Draggable board objects */
```

### Layer 2: JavaScript Runtime Hooks (cursor-authority.js)
**Purpose:** Reactive cursor updates when Foundry dynamically changes cursors via JS
**Coverage:** Tool state changes, canvas interactions, Pixi events

**Mechanisms:**
1. **Event Hooking:** Listen to Foundry's `ready`, `canvasReady` hooks
2. **DOM Events:** Attach `mouseenter`, `mouseleave`, `dragstart`, `dragend` handlers
3. **Pointer Events:** Track `pointerdown`, `pointerup`, `pointermove` on canvas
4. **Mutation Observer:** Detect if external code overwrites cursor style
5. **Emergency Reset:** Force reset all cursors if corruption detected

**Key Additions:**
```css
/* Main scene canvas */
body.game #board { cursor: var(--swse-cursor-default); }

/* Placeables (tokens, walls, templates) */

/* Draggable board elements */
body.game [draggable="true"].placeable { cursor: var(--swse-cursor-grab); }
body.game [draggable="true"].placeable:active { cursor: var(--swse-cursor-grabbing); }

/* Locked/forbidden items */
body.game .placeable[data-locked="true"] { cursor: var(--swse-cursor-forbidden) !important; }
```

**Scoping:** All rules prefixed with `body.game` to ensure they only apply in the game view, not character creation/other contexts

---

### 2. `/scripts/ui/cursor-authority.js` (NEW FILE)
**Purpose:** Runtime cursor enforcement system
**Size:** 250+ lines

**Key Classes & Methods:**

#### `CursorAuthority` Static Class
- `initialize()` — Set up all hooks on system init
- `applyBoardCursor(cursorType)` — Apply SWSE cursor to board element
- `_hookToolEvents()` — Listen for tool changes
- `_hookCanvasEvents()` — Attach DOM event listeners
- `_observeCursorChanges()` — MutationObserver for style overrides
- `forceReset()` — Emergency reset of all cursors
- `debugCursorState()` — Log current cursor state (dev tool)

**Initialization Flow:**
1. On Foundry `init` hook, call `CursorAuthority.initialize()`
2. Apply initial board cursor ('default')
3. Attach event hooks to Foundry canvas
4. Start observing for external cursor overrides
5. Ready to handle dynamic cursor changes

**Event Binding:**
- `canvasReady` → Apply default board cursor
- `mouseenter` on board → Apply default cursor
- `mouseleave` on board → Apply default cursor
- `dragstart` on board → Apply grabbing cursor
- `dragend` on board → Apply default cursor
- `pointerdown` on canvas → Apply grabbing cursor
- `pointerup` on canvas → Apply default cursor
- `pointermove` on canvas → Apply default cursor

**Mutation Observation:**
- Watches for style attribute changes on board
- If cursor overridden by external code, immediately reapplies SWSE cursor
- Logs diagnostic message when override detected

## Technical Details

### CSS Variable Resolution
The cursor authority JS reads CSS custom properties dynamically:
```javascript
const varName = this.CURSOR_VARS[cursorType]; // e.g., '--swse-cursor-default'
const cursorValue = getComputedStyle(document.documentElement)
  .getPropertyValue(varName)
  .trim();
boardEl.style.cursor = cursorValue;
```

This ensures cursors are always in sync with the CSS variable definitions.

### Cursor Type Mapping
```javascript
CURSOR_VARS = {
  default:   '--swse-cursor-default',      // General UI
  hover:     '--swse-cursor-hover',        // Clickable elements
  select:    '--swse-cursor-select',       // Selected/active items
  grab:      '--swse-cursor-grab',         // Draggable contexts
  grabbing:  '--swse-cursor-grabbing',     // Active drag
  precision: '--swse-cursor-precision',    // Fine control
  text:      '--swse-cursor-text',         // Text input
  forbidden: '--swse-cursor-forbidden',    // Disabled/locked
}
```

### Fallback Chain
All cursor rules include system fallback cursors:
```css
--swse-cursor-grab: url('...') 24 18, grab;     /* SVG, fallback to 'grab' */
--swse-cursor-forbidden: url('...') 24 24, not-allowed;  /* Fallback to 'not-allowed' */
```

If SVG asset loads, user sees custom cursor. If asset fails, browser shows system cursor. No broken state.

## Known Limitations

1. **Pixi Rendering:** If Foundry's Pixi layer explicitly sets canvas.style.cursor in JavaScript, it will override CSS until the next event/observer cycle. Cursor will snap back to SWSE within ~50ms.

2. **Tool Overrides:** Some tools may set cursor directly. The MutationObserver will detect and reapply SWSE cursor, but there may be brief flicker.

3. **SVG Asset Loading:** If cursor SVG assets fail to load (404, CORS), cursors fall back to system cursors. This is intentional — better to have working system cursor than broken state.

4. **Accessibility:** Some users disable custom cursors. System respects `prefers-reduced-motion` and high contrast mode; SWSE cursors still apply but with appropriate system fallbacks.
